The Problem

Trip planning on Telegram is fragmented; links are scattered across messages, and people waste too much time manually consolidating places, stays, and activities.

The solution

Meet users where they already coordinate: inside Telegram, not another standalone planner app.

This product direction prioritizes low-friction adoption by embedding planning tools into existing group-chat behavior. Instead of asking users to migrate tools, the bot turns current chat activity into structured travel planning artifacts.

Chapter 01 · Problem Framing

Planning breaks in the chat

Young adults, especially university students, often plan trips on Telegram. Telegram does not have specialised features for trip planning: links and comments get buried, and the group wastes time scrolling and manually reorganising information.

The messy and time-consuming collaboration faced opens up an opportunity to introduce structure without changing user behaviour or platform. The solution needed to work in the same place conversations already happen.

Chapter 02 · Solution Overview

A command-led bot with LLM summarisation

The solution involves a Telegram bot that optimises the trip-planning experience by turning unstructured trip chat into a usable planning view without requiring users to leave their existing conversation flow.

At a high level, it captures group intent, organising shared links into clear categories, and returns a concise AI-generated trip summary with map-based context.

Chapter 03 · User Flow

From chat signals to a group-ready plan

1. Start the flow: The bot can be added at any point in the group conversation and activated with /start.

2. Capture context: A short Q&A collects trip intent, constraints, and preferences before links are processed.

3. Gather and confirm links: Shared links are collated and surfaced via /links, with an option to remove irrelevant entries before summarisation.

4. Analyse shared content: Link content is parsed to extract destinations, points of interest, and themes into machine-readable inputs.

5. Generate summary output: Structured inputs are sent to the LLM to produce a concise, group-ready planning summary.

6. Iterate with new inputs: Running /plan regenerates the summary as new links arrive, while preserving useful context when still valid.

Chapter 05 · Implementation of Map

Mini app map for richer interactions

Work Flow

  1. Telegram bot sends the map link (rendered in a webpage) after the summary output.
  2. Clicking the link guides users to the webpage, either in Telegram via Mini App or in a web browser.
  3. The map is live and interactive, allowing users to view pinned locations and click pins for more details.

Technical Implementation Plan for Telegram Mini App

Telegram's inbuilt UI restricts custom markers. Shifting the map into a mini app to load markers based on chat is the proposed workaround.

The mini app provides the following functionalities:

  1. Info Cards when pins are clicked
  2. Different Colours based on Different Categories
